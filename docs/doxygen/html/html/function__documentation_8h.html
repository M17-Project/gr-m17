This function performs Curve25519 ECDH key agreement to establish a shared secret with a peer. The shared secret can then be used for symmetric encryption.

◆ set_m17_strict_mode()

void set_m17_strict_mode ( bool  strict_mode)

Set M17 strict mode.

Parameters
strict_modeEnable strict M17 compliance

This function enables or disables M17 strict mode. In strict mode, only M17 specification compliant features are used (P-256 ECDSA, AES-256-CTR). This ensures maximum compatibility with other M17 implementations.

Compatibility Modes

Strict Mode (true):

  • P-256 ECDSA for digital signatures
  • AES-256-CTR for encryption
  • Standard LSF/META field usage
  • Compatible with all M17 implementations

Extended Mode (false):

  • Ed25519 digital signatures
  • Curve25519 ECDH key agreement
  • AES-256-GCM authenticated encryption
  • Nitrokey HSM integration
  • NOT M17 spec compliant
Warning
SECURITY WARNING: Extended mode is NOT M17 spec compliant
SECURITY WARNING: Extended mode requires coordination with other stations
Note
Use strict mode for maximum compatibility
Extended mode is for advanced users with Nitrokey hardware

Usage Examples

// Enable strict M17 mode (default, recommended)
decoder.set_m17_strict_mode(true);
// Enable extended mode (requires coordination)
decoder.set_m17_strict_mode(false);
decoder.set_extended_crypto(true);
Parameters
strict_modeEnable strict M17 compliance
Returns
void
See also
set_extended_crypto() for extended mode
is_m17_compatible() for compatibility check
